## 2026-04-09 - Defer RNG for Stationary Drops
**Learning:** In the digital rain render loop, drops with a low falling velocity (e.g. `vY` large, acting as a frame delay via `frameCounter % vY == 0`) remain stationary for several frames. Previously, the code blindly generated a new random character and invoked `updateChar` on every frame, even when the drop hadn't moved. This wasted CPU cycles on the custom LCG and forced unnecessary array reads/writes.
**Action:** When iterating over objects with fractional/delayed movement, evaluate the movement delta first. Only trigger dependent state calculations (like random appearance changes or trail updates) and array writes if the position has actually changed (`pXN != pXC || pYN != pYC`).
